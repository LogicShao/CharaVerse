## 项目概述

### 项目名称
CharaVerse：OC 管理系统与名片展示前端项目

### 技术栈
- **前端框架**：React 18+ with TypeScript
- **状态管理**：Zustand 或 Context API
- **样式方案**：CSS Modules 或 Tailwind CSS
- **数据存储**：本地 JSON 文件或 LocalStorage
- **构建工具**：Vite

## OC JSON 数据模板结构（11个部分）

### 1. Basic Profile（基础信息块）
- 唯一标识符（UUID）
- 英文名和中文名
- 昵称和别称
- 创建日期和最后修改日期
- 创作者信息
- Schema 版本号

### 2. Appearance（外观设计块）
- 身体信息（年龄、身高、体重、体型、肤色）
- 面部特征（脸型、眼睛、眉毛、鼻子、嘴唇、脸部标记）
- 发型信息（发色、发长、发型、发质）
- 其他外观（体毛、装饰品、疤痕、纹身）

### 3. Wardrobe（服装与配饰块）
- 默认服装
- 多套服装变化（名称、描述、颜色、风格、关联故事）
- 主要配饰列表
- 标志性武器或工具

### 4. Personality & Psychology（性格与心理块）
- 性格类型（MBTI、星座、血型）
- 性格特征（核心描述、5-7个主要特质、性格动机）
- 心理状态（主要恐惧、核心欲望、创伤、心理防御机制）
- 表现方式（说话习惯、口头禅、行动风格、情感表达）

### 5. Background Story（背景故事块）
- 出生地和社会背景
- 家庭成员和关系
- 童年经历
- 重要的人生转折点
- 教育背景
- 主要经历和冒险
- 当前生活状态

### 6. Skills & Abilities（能力与专长块）
- 职业或身份
- 主要技能列表（附带等级或熟练度）
- 特殊能力（如魔法、超能力等）
- 弱点和限制
- 战斗风格（如果适用）
- 工具掌握度

### 7. Relationships（关系网络块）
- 与其他 OC 或既有角色的关系
- 关系对象名称、关系类型、关系强度、关系描述
- 重要事件或共同回忆

### 8. Lore & World（设定与世界块）
- 角色所属的宇宙或世界观
- 种族或物种信息
- 社会阶级或身份地位
- 文化或派系从属
- 宗教或信仰
- 价值观和生活哲学

### 9. Additional Info（补充信息块）
- 爱好和兴趣
- 讨厌的东西
- 特殊习惯
- 梦想和目标
- 座右铭或个性签名
- 标签（用于分类和检索）

### 10. Media Assets（媒体资源块）
- 头像图片 URL/本地路径
- 全身图片 URL/本地路径
- 其他参考图片列表
- 多个角度的视图链接
- 官方配图或粉丝艺术

### 11. Metadata（元数据块）
- 创建者信息
- 版本历史
- 许可证信息
- 相关链接（AO3、微博、推特等）
- 自定义标签

## TypeScript 类型系统设计

### 分层结构
1. **第一层**：原始类型和枚举
2. **第二层**：单一职责接口
3. **第三层**：组合接口
4. **第四层**：工具类型和泛型

### 关键接口
- `BasicProfile`：基础信息
- `AppearanceProfile`：外观信息
- `PersonalityProfile`：性格心理
- `RelationshipEntry`：单个关系
- `CompleteOCProfile`：完整角色数据结构

## 前端名片展示组件设计

### 主要组件
1. **OCCardPreview**：卡片预览组件（用于列表展示）
2. **OCCardComponent**：完整名片卡片（独立可复用）
3. **AppearancePanel**：外观信息面板
4. **PersonalityPanel**：性格信息面板
5. **RelationshipsPanel**：关系面板
6. **BackgroundPanel**：背景故事面板
7. **SkillsPanel**：技能面板
8. **GalleryPanel**：媒体资源面板

### 页面布局
- **OCListPage**：列表页面（搜索、筛选、卡片网格）
- **OCDetailPage**：详情页面（多面板展示）

## 数据流和状态管理

### 状态管理工具
- **推荐**：Zustand（轻量且易上手）

### 状态结构
- **OCStore**：管理所有 OC 相关的状态和操作方法
- **UIStore**：管理 UI 相关的状态
- **SettingsStore**：管理用户设置

## 开发工作流

### 四个阶段
1. **类型系统建设**：定义 TypeScript 接口和类型
2. **组件开发**：从基础 UI 组件到数据展示组件
3. **功能集成**：数据加载、搜索、编辑、导入导出
4. **优化和测试**：性能优化、单元测试、响应式设计

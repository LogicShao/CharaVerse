# CharaVerse 项目文档访问指南

## 📚 文档体系结构

### 核心文档位置
```
项目根目录/
├── .claude/                    # AI 记忆优化目录
│   ├── CLAUDE.md              # 主开发指南（自动加载）
│   ├── IMPLEMENTATION_PLAN_SUMMARY.md    # 实施计划摘要
│   ├── COMPLETED_STAGES_SUMMARY.md       # 已完成阶段摘要
│   ├── PROJECT_SUMMARY.md                # 项目大纲摘要
│   └── DOCUMENTATION_GUIDE.md            # 本文档
│
├── IMPLEMENTATION_PLAN.md     # 完整实施计划（10阶段）
├── COMPLETED_STAGES.md        # 阶段完成记录
├── PROJECT.md                 # 详细项目设计文档
└── README.md                  # 项目说明
```

## 🎯 智能访问策略

### 1. 日常开发参考
**使用摘要文件**（`.claude/*_SUMMARY.md`）：
- 快速了解当前阶段任务
- 查看验收标准
- 获取核心信息

### 2. 详细规划时
**读取完整文档**（根目录文件）：
- 开始新阶段前
- 需要完整任务清单时
- 核对详细设计时

### 3. CLAUDE.md 自动记忆
**每次对话自动加载**：
- 开发原则和规范
- 项目结构说明
- 常用命令
- 当前状态摘要

## 💡 最佳实践

### 对于 AI（Claude Code）
1. **对话开始时**：自动获取 CLAUDE.md 中的开发指南
2. **执行任务前**：查看相关摘要文件了解上下文
3. **需要详细信息时**：读取完整文档的特定部分
4. **更新状态时**：同步更新摘要文件

### 对于开发者
1. **日常开发**：参考 CLAUDE.md 和摘要文件
2. **规划阶段**：阅读完整实施计划
3. **设计功能**：参考完整项目设计文档
4. **记录进度**：更新完成阶段记录

## 🔄 同步维护

### 更新流程
1. **完成阶段时**：
   - 更新 `COMPLETED_STAGES.md`
   - 更新 `.claude/COMPLETED_STAGES_SUMMARY.md`
   - 更新 `.claude/CLAUDE.md` 中的当前状态

2. **修改计划时**：
   - 更新 `IMPLEMENTATION_PLAN.md`
   - 更新 `.claude/IMPLEMENTATION_PLAN_SUMMARY.md`

3. **设计变更时**：
   - 更新 `PROJECT.md`
   - 更新 `.claude/PROJECT_SUMMARY.md`

### 版本控制
- 所有文档都纳入 Git 版本控制
- 摘要文件随主文档一起更新
- 保持摘要与完整文档的一致性
